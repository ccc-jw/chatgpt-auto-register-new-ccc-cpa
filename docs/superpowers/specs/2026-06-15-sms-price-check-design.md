# 短信价格检查设计

## 背景

当前 Web GUI 已支持配置短信平台、国家/地区购买池、最高价格，并可以按价格排序使用国家。新增“短信价格检查”用于查看当前短信平台 OpenAI 服务在全部国家/地区的价格分布，并结合本地历史成功价格统计，判断哪些国家/地区值得加入购买池。

## 目标

- 在 Web GUI 的“下载结果”和“余额”之间新增“短信价格检查”入口。
- 点击入口跳转到独立页面 `/sms-price-check`。
- 查询当前短信平台 OpenAI 服务返回的全部国家/地区价格。
- 展示价格检查表，支持每列筛选。
- 新增成功价格统计表，注册手机号阶段成功时聚合记录。
- 推荐逻辑同时考虑低价和历史成功价格统计。
- 不暴露短信 API key、SUB2API 密码、邮箱密码等敏感配置。

## 非目标

- 不自动修改 `config.json` 的 `sms.countries`。
- 不在本次功能中自动购买或批量测试号码。
- 不依赖短信平台提供成功率接口。
- 不把运行统计数据提交到 git。

## 页面入口与布局

现有 Web GUI 顶部操作区增加按钮，顺序为：

1. 下载结果
2. 短信价格检查
3. 余额

点击“短信价格检查”打开 `/sms-price-check`。页面复用现有 Web GUI 的基础样式，包含两个表格。

### 价格检查表

列字段：

- 短信平台
- 国家/地区 ID
- 最低价格
- 倒数第二低价格
- 最低价格数量
- 倒数第二低价格数量
- 是否已在购买池
- 是否推荐加入购买池

行为：

- 每一列提供筛选输入或选择器。
- 价格列支持按数值排序。
- 推荐列显示可解释文本，而不是只显示布尔值。

### 成功价格统计表

列字段：

- 短信平台
- 国家/地区 ID
- 成功价格
- 成功次数

行为：

- 每一列提供筛选输入。
- 成功次数支持按大小排序。
- 表格数据来自本地成功价格统计文件。

## 后端接口

新增价格检查接口，供 `/sms-price-check` 页面加载数据。

输入来源：

- 当前 `config.json` 的短信平台、API key、`sms.countries`、`sms.max_price`。
- 当前短信平台 OpenAI 服务价格接口。
- 本地成功价格统计文件。

输出内容：

- `price_rows`：价格检查表行数据。
- `success_rows`：成功价格统计表行数据。
- `configured_countries`：当前购买池国家/地区 ID 列表。
- `errors`：非致命错误信息。

接口响应不得包含 API key、SUB2API 密码、邮箱密码、session token 或 refresh token。

## 价格解析

服务固定查询 OpenAI，即当前代码中使用的 `dr` 服务。

对每个国家/地区，从平台返回的 provider 价格集合中计算：

- 最低价格：所有可用 provider 价格中的最小值。
- 倒数第二低价格：去重后的第二低价格；如果不存在则为空。
- 最低价格数量：价格等于最低价格的 provider 数量。
- 倒数第二低价格数量：价格等于倒数第二低价格的 provider 数量。

某个国家/地区解析失败时，该国家/地区标记为不可用，其他国家继续展示。

## 成功价格统计

新增本地统计文件 `sms_success_prices.json`，并加入 `.gitignore`。

注册流程中，当结果满足 `phone_ok=true` 时记录一次成功价格。记录维度：

- 短信平台
- 国家/地区 ID
- 成功价格

聚合规则：

- 如果相同 `短信平台 + 国家/地区 ID + 成功价格` 已存在，则 `成功次数 + 1`。
- 如果不存在，则新增一行，`成功次数 = 1`。

成功价格来源：

- 优先使用短信平台在接收短信成功链路中返回或保存的实际价格。
- 如果平台没有实际成交价字段，则使用拿号前该国家/地区查询到的最低价格作为兜底价格。
- 只有 `phone_ok=true` 时才写入统计，Phase2、SUB2API 或 CPA 上传失败不影响短信成功统计。

统计写入失败只记录错误日志，不阻断注册主流程。

## 推荐规则

推荐结果以可解释文本展示：

- 已经在 `sms.countries` 里的国家/地区：显示“已在池中”。
- 没有可用价格：显示“不推荐：无可用价格”。
- 最低价格高于 `sms.max_price`：显示“不推荐：高于最高价格”。
- 最低价格不高于 `sms.max_price`，且有历史成功统计：显示“推荐：低价 + 历史成功 N 次”。
- 最低价格不高于 `sms.max_price`，但没有历史成功统计：显示“可试用：低价但无历史成功”。

推荐排序优先考虑：

1. 是否已在购买池。
2. 是否低于或等于最高价格。
3. 历史成功次数。
4. 历史成功价格。
5. 当前最低价格。

页面不自动把推荐国家写入购买池，用户手动决定是否更新配置。

## 错误处理

- 未配置短信 API key：价格检查页面显示明确错误，历史成功统计表仍可展示。
- 平台价格接口失败：展示错误信息，不修改统计文件。
- 部分国家价格解析失败：只影响对应行。
- 成功统计写入失败：记录日志，不影响注册结果返回。
- 所有 API 响应都不返回敏感配置值。

## 测试计划

- 价格解析单元测试：最低价、倒数第二低价、数量统计、空价格集合。
- 成功统计单元测试：相同平台/国家/价格累加，不同价格分行。
- 推荐规则单元测试：已在池中、低价有历史、低价无历史、高于最高价格、无可用价格。
- Web GUI 测试：入口按钮存在、新页面存在、两个表格列完整、筛选控件存在、成功次数可排序。
- 敏感字段测试：价格检查接口响应不包含 API key、密码、token。
